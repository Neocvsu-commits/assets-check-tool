# 资产审查助手（团队内部使用文档）

## 1. 插件定位
资产审查助手用于在模型提交和导出前，快速发现并修复影响引擎落地的常见问题。你可以把它理解为“统一验收 + 快速修复 + 报告归档”的检查入口。

它的核心目标是：

- 让检查项可配置、可复用（预设化管理）。
- 让高频问题可定位、可一键修复（减少手工排查）。
- 让检查结果可追踪（矩阵视图 + CSV/JSON 导出）。
- 与资产导出助手联动，保证审查结果可跟随资产一起交付。

## 2. 基本信息

- 插件版本：`2.0.0`（正式版）
- Blender 支持：`4.2+`
- 入口：`3D 视图 > 顶栏 > 检查`

## 3. 典型工作流

### 场景 A：外包回收资产验收
1. 选中回收资产，加载“外包验收”预设。
2. 执行检查并先清理 `FAIL`。
3. 对 `WARN` 做人工复核（是否可豁免）。
4. 导出 CSV/JSON 作为返修依据。

结果：你得到可追踪的返修清单，而不是口头反馈。

### 场景 B：引擎导出前质检
1. 选中待导出对象并执行检查。
2. 使用列菜单快速处理 `N` 边面、法线、UV、轴心、碰撞等问题。
3. 复检通过后再进入资产导出助手做统一导出。

结果：导出后在引擎里的返工概率会明显下降。

### 场景 C：团队统一标准落地
1. 由负责人配置检查组合并保存为预设。
2. 将预设文件导出给团队成员导入。
3. 每次提交前按同一预设检查并导出报告。

结果：同一项目内的审查口径一致，减少“每个人标准不同”的争议。

## 4. 标准使用流程（推荐）

1. 在视图中选中要审查的 `MESH` 物体。
2. 点击顶栏 `检查` 打开资产审查弹窗。
3. 在“自定义检查”中选择或加载预设，确认要启用的检查项。
4. 点击 `开始检查`。
5. 在结果矩阵中优先处理 `FAIL`，再处理 `WARN`。
6. 需要批量修复时用 `一键修复`；需要精修时用每列菜单中的定位/修复工具。
7. 点击 `导出报告 (CSV、JSON)` 输出审查结果归档。

## 5. 检查与修复机制

### 4.1 检查项分组
当前版本将检查项按业务维度拆为以下组：

- 材质与贴图：空材质槽、贴图丢失
- UV 与颜色：越界、重叠、UV 数量、顶点色数量、`UV0` 豁免
- 拓扑与几何：`N` 多边面、非流形、游离几何、重叠顶点、极点、法线、不平整面、零边、交叉边面
- 物体与数据：缩放未应用、变换未归零、轴心位置、修改器、动画、顶点权重、UE 简易碰撞
- 命名规范：物体/材质/贴图命名前缀规则

### 4.2 结果视图
- 结果以矩阵方式展示：行是物体，列是检查项。
- 支持按名称搜索、按列排序、按结果选中对象。
- 每一列提供快捷下拉菜单，可直接执行对应定位或修复操作。

### 4.3 一键修复与定向修复
- `一键修复` 处理高频、无争议的基础问题（如空材质槽、游离几何、重叠点、基础变换）。
- 每列菜单提供定向工具，例如：
  - `定位越界UV`、`定位重叠UV`
  - `纯三角化N-gon`
  - `定位反转法线`
  - `应用缩放`、`应用全部变换`
  - `自动生成凸包碰撞`

### 4.4 预设机制
- 支持保存、删除、导入、导出预设。
- 默认预设为“全部检查项开启”。
- 推荐按项目类型维护预设，例如高精资产、移动端资产、外包验收资产。

## 6. 报告与联动

### 5.1 审查报告输出
点击 `导出报告 (CSV、JSON)` 后会输出：

- `*.csv`：可直接给 Excel 或项目管理平台使用
- `*.json`：用于工具链读取或自动化流程

报告每行包含对象名、检查项、状态和提示信息，便于复查和返修跟踪。

### 5.2 与资产导出助手联动
- 资产导出助手可读取本插件的审查结果并导出 `Check` 报告。
- 如果当前导出对象未完成审查，导出端对应审查选项会灰显或被跳过。
- 建议在导出前先完成本插件的检查，避免交付包缺少审查记录。

## 7. 默认检查开关（当前）

当前默认策略为：检查项全部开启。  
这保证首次安装即可覆盖团队常用风险点；之后可按预设精简。

## 8. 限制与边界情况（查表）

| 项目 | 现象 | 原因 |
| :--- | :--- | :--- |
| 弹窗被意外关闭 | 点击弹窗外部后面板消失 | 当前采用弹窗交互，关闭后需要从顶栏 `检查` 重新打开 |
| 一键修复后几何数量变化 | 顶点/边/面数量减少 | 清理游离几何、合并重叠点本质是数据删减，属于预期行为 |
| UV 重叠有时不报 | 不同材质复用同一区域未告警 | 检查逻辑按材质槽隔离，跨材质复用 `[0,1]` 不视为同类重叠 |
| 命名检查与旧习惯不一致 | 旧资产批量报命名问题 | 当前命名规则对齐 `SM_` / `M_` / `T_` 前缀标准 |
| 导出助手里审查项灰色 | 审查 CSV/JSON 不能勾选 | 导出端要求当前选中对象已有有效审查结果，避免导出空报告 |
| 定位交叉面执行失败 | 某些版本报算子参数错误 | Blender 4.x 的 `mesh.intersect` 参数有变更，插件已按新枚举修正；旧脚本缓存需重载 |
